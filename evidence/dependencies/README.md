# Python Dependency Evidence

This directory contains Python dependency evidence for the Mini SILS/HILS PoC.

The existing baseline was evaluated and accepted under the evaluation conditions defined at the completion of Phase ②.

In Phase ③-A, Python dependency evidence was added as supplementary evidence to the existing baseline.

## Purpose

The purpose of this evidence is to record which Python environment was used when the current PoC baseline was handled.

This is the first step toward connecting the following items in later phases:

- Dependency evidence
- SBOM (Software Bill of Materials)
- Evidence Manifest
- CI (Continuous Integration) artifacts
- Governance checks
- Container or execution environment evidence

## Files

- `python_freeze.txt`
  - Snapshot of the current Python virtual environment generated by `pip freeze`.

- `../sbom/python_environment_sbom.json`
  - CycloneDX SBOM generated from the current Python virtual environment.

## Current Scope

This phase records the current Python virtual environment as-is.

The generated SBOM represents the development and execution Python environment at the time of generation.

It is not yet separated into:

- runtime-only SBOM
- development-tool SBOM
- container SBOM

## Deferred Items

The following items are deferred to later phases:

- Connecting SBOM to `evidence/evidence_manifest.json`
- Uploading SBOM and dependency evidence as GitHub Actions artifacts
- Adding governance checks for missing or outdated SBOM evidence
- Creating container metadata
- Generating container SBOM
- Separating runtime dependencies from development and evidence-generation dependencies

## Notes

This PoC uses only fictional requirements, scenarios, and evidence.

No company information, customer information, real project information, real vehicle data, internal documents, or customer-specific data are included.
