# Phase 29 Artifact and Evidence Architecture

## Purpose

This document defines responsibilities for evidence generation, artifact retention, and AWS S3 storage.

The goal is to separate:

* evidence generation
* evidence retention
* evidence storage

## Current State

### Python SILS CI

Responsibilities:

* pytest execution
* governance checks
* evidence manifest generation
* coverage matrix generation

Generated evidence:

* evidence_manifest.json
* coverage_matrix.json

### C++ SILS CI

Responsibilities:

* C++ build
* C++ unit tests
* SILS suite execution
* Python/C++ comparison

Generated evidence:

* comparison_summary.json

### AWS S3 Upload Workflow

Responsibilities:

* upload approved evidence to S3

Currently uploads:

* verification_report.md
* evidence_manifest.json
* comparison_summary.json
* coverage_matrix.json

## Identified Gap

verification_report.md is not currently generated by CI.

Artifact contents are not aligned with S3 upload contents.

## Target Architecture

Python SILS CI
→ governance evidence

C++ SILS CI
→ comparison evidence

Artifact
→ official evidence package

AWS Upload Workflow
→ artifact retrieval
→ S3 storage
