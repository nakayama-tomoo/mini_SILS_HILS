# Lifecycle Work Products

This directory organizes lifecycle work products for the Mini-SILS-HILS PoC.

The purpose is to connect:

- requirements analysis
- design
- implementation
- evaluation
- verification
- validation
- traceability
- evidence management

This project is a personal PoC.

Do not use:

- company data
- customer data
- real vehicle data
- confidential documents
- customer-specific requirements
- company project source code
- actual ECU logs
- real vehicle connection

## Directory Policy

### 00_project

Project scope, restructuring plan, assumptions, and roadmap.

### 10_requirements_analysis

Needs, use cases, constraints, system requirements, software requirements, evaluation requirements, SQA requirements, and acceptance criteria.

### 20_design

Architecture, interfaces, data models, environment models, test design, and safety/cost/license notes.

### 30_implementation

Mapping between design items and implementation assets.

### 40_evaluation

Evaluation plans, KPI definitions, environment selection policy, and result interpretation notes.

### 50_verification

Verification plans, test cases, requirement verification matrices, coverage policy, and verification reports.

### 60_validation

Validation plans, validation scenarios, feasibility assessment, acceptance review, and validation reports.
